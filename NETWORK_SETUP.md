# 手机/平板访问设置指南

## 目标
让手机、平板可以通过网络访问电脑上的"小习惯管家"，并使用全部功能（包括语音提醒）。

## 前提条件
1. 手机和电脑必须在**同一个WiFi网络**下
2. 电脑需要开启本地服务器

## 设置步骤

### 1. 查找电脑的局域网IP地址

**Windows:**
```bash
# 打开命令提示符，输入：
ipconfig

# 查找 "IPv4 地址"，例如：192.168.1.100
```

**Mac:**
```bash
# 打开终端，输入：
ifconfig | grep "inet "

# 查找类似 192.168.x.x 的地址
```

### 2. 启动服务器（绑定所有网络接口）

当前服务器只绑定了 localhost，需要修改为绑定所有IP：

```bash
# 在当前目录下执行：
python -m http.server 8080 --bind 0.0.0.0
```

或者使用我提供的启动脚本：

```bash
# Windows
start-server.bat

# Mac/Linux
./start-server.sh
```

### 3. 手机访问

在手机浏览器中输入：
```
http://[电脑IP地址]:8080

# 例如：
http://192.168.1.100:8080
```

## 注意事项

### 1. 防火墙设置
如果手机无法访问，可能是防火墙阻止了8080端口：

**Windows:**
1. 打开"Windows Defender 防火墙"
2. 点击"允许应用通过防火墙"
3. 添加 Python 或 8080 端口

### 2. 语音播放
- 手机浏览器需要允许**自动播放音频**
- 首次访问时可能需要用户交互（点击页面）后才能播放语音
- 建议在手机浏览器设置中允许该网站播放音频

### 3. 保持屏幕常亮
为了确保语音提醒能正常触发，建议：
- 手机：设置屏幕常亮或延长自动锁屏时间
- 保持浏览器在前台运行

## 功能一致性

手机/平板访问时，所有功能与电脑一致：
- ✅ 创建/编辑/删除任务
- ✅ 语音提醒（使用系统TTS）
- ✅ 倒计时功能
- ✅ 星星奖励系统
- ✅ 本地数据存储（各设备独立）

## 数据同步说明

**注意：** 数据存储在各自设备的浏览器中，电脑和手机的数据是独立的。

如需同步数据，可以使用"导出数据"和"导入数据"功能。

## 故障排查

### 手机无法访问
1. 检查电脑和手机是否在同一WiFi
2. 检查防火墙设置
3. 尝试关闭电脑防火墙测试
4. 检查IP地址是否正确

### 语音无法播放
1. 确保手机音量开启
2. 浏览器需要允许音频播放
3. 首次播放需要用户点击页面
4. 检查"语音总开关"是否开启

### 页面显示异常
1. 清除浏览器缓存
2. 使用Chrome/Safari等现代浏览器
3. 确保网络连接稳定
